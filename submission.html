Question 1

Point of Indifference

The geographic point of indifference is the point along our 1 dimensional space at which customers have no preference for one hospital over the other. Customers to the “right” of this point will be happier patronizing the hospital further to the right, and customers to the “left” of this point will be happier patronizing the hospital further to the left. Therefore, calculating this point of indifference is tantamount to calculating the proportion of the total customers that either company will claim. This indifference point is a function of the price that Hospital 1 charges (\(p_{1}\)) and the price that Hospital 2 charges (\(p_{2}\)).

For our purposes, a single customer's happiness with regards to a particular hospital \(i\) is defined as: \[10 - 2\cdot (x_{i})^{2} - p_{i}\] where \(p_{i}\) is the price that they choose to pay (of the two offered), \(x_{i}\) is the distance from the customer to the hospital charging that price. The point of indifference is the point at which this metric is equal for both hospitals. If \(L\) is the point of indifference: \[10 - 2\cdot (L-x_{1})^{2} - p_{1} = 10 - 2\cdot (L-x_{2})^{2} - p_{2}\] Solving \(L\) in terms of \(p_{1}\), \(p_{2}\), \(x_{1}\), and \(x_{2}\): \[L = x_{indif}=\frac{(2\cdot x_{2}^{2})-(2\cdot x_{1}^{2})+p_{2}-p_{1}}{(4\cdot x_{2}-4\cdot x_{1})}\] This equation makes intuitive sense because \(x_{indif}\) is pulled closer to Hopsital 1 if it raises its price \(p_{1}\), reducing is range of appeal to customers, while \(x_{indif}\) is similarly pulled closer to Hospital 2 if it raises its price \(p_{2}\).

Simulation

We are now armed with most of what we need to determine where the optimal location is for the firms if they wish to maximize profits, which we will accomplish via simulation. One additional piece of information our simulation will require is the physical distribution of customers over our unidimensional landscape. We will first run simulations that assume a uniform distribution \(Unif(0,1)\), and then rerun the simulations assuming a beta distribution \(Beta(2,5)\).

Overview

  • First we loop overa grid of possible combination of hospital locations. For each:
    • Loop over every possible combination of hospital prices (binned in increments of .25 from 0 to 10). For each:
      • Calculate the indifference point using the equation derived previously, based on the \(p_{1}\) and \(p_{2}\) of the current iteration.
      • Use our distribution \(\mathcal{U}(0,1)\) or \(Beta(2,5)\), together with the indifference point, to calculate the number of customers who will prefer Hospital 1 and the number of customers who will prefer Hospital 2.
      • Multiplying this results by each Hospital's respective price and adjusting for our assumed marginal cost of 0.5 returns to us the simulated profit \(\pi\) of each company at the present combination of \(p_{1}\) and \(p_{2}\).
      • The indifference point splits the population into two segments. In order to calculate the happiness of the community, we determine the expected location of a citizen in each segment by using the CDFs of the distributions. Taking these expected locations and subtracting the closest firm's location gave the expected distance to travel for each cohort. Total happiness is then simply the sum of two happiness equations.
      • We now have a grid of \(p_{1}\) and \(p_{2}\) and the profits of both hospitals for each.
      • Perform a simulation by randomly seeding \(p_{1}\) and \(p_{2}\). For each step in this iterative simulation:
        • Each hospital adjusts its price so as to maximize its profit while operating under the constraint imposed by the other hospital's current price.
        • Eventually this simulation converges to a set of prices at which neither hospital has an incentive to change their price (the “equilibrium prices”).
      • Recall that this is only the equilibrium price for our current given hospital locations (the outermost loop in the algorithm).
      • We now append this set of equilibrium prices, profits, and total happiness to a running list.
  • Lastly, we implement a similar turn-taking simulation, this time with randomly initialized hospital locations.
  • We allow each hospital to update its location so as to maximize their profit while operating under the constraint of the other hospital's current location
  • Eventually this simulation converges to a set of locations at which neither hospital has an incentive to change their location (the “equilibrium locations”).

We need to define a few helper functions in order to simulate customers from these distributions to make our code reusable between arbitrary customer distributions. The first function cdf_citizen uses the indifference point to calculate the proportion of customers that prefer each hospital. The second function takes this proportion and returns the absolute position of the given cohorts expected location, this can then be transformed to give the average distance from customer to preferred hospital.

cdf_citizen <- function(distro, indif) {
						if (distro == 'beta') {
						pbeta(indif, 2, 5)
					} else {
					punif(indif, 0, 1)
				}
			}

			q_citizen <- function(distro, p) {
			if (distro == 'beta') {
			qbeta(p, 2, 5)
		} else {
		qunif(p, 0, 1)
	}
}

We define a function run_sim that simulates profits and happiness for a given combination of \(x_{1}\) and \(x_{2}\).

run_sim <- function(distro, x1, x2) {

	d <- data.frame('p1'=NULL, 'p2'=NULL, 'type'=NULL, 'pi'=NULL, 'hap'=NULL)

	# Loop over price combinations
	for (p1 in seq(0, 10, price_step)) {
	for (p2 in seq(0, 10, price_step)) {

	# Assign citizens to a hospital based on who they will be happiest with
	indif <- max(0, min(1, ((2*x2^2) - (2*x1^2) + p2 - p1) / (4*x2 - 4*x1)))

	if (x1 < x2) {
	cohort1 <- cdf_citizen(distro, indif)
	cohort2 <- 1 - cdf_citizen(distro, indif)
	dist1 <- abs(x1 - q_citizen(distro, cohort1/2))
	dist2 <- abs(x2 - q_citizen(distro, ((1 - cohort1)/2) + cohort1))
} else if (x1 > x2) {
cohort1 <- 1 - cdf_citizen(distro, indif)
cohort2 <- cdf_citizen(distro, indif)
dist1 <- abs(x1 - q_citizen(distro, ((1 - cohort1)/2) + cohort1))
dist2 <- abs(x2 - q_citizen(distro, cohort1/2))
} else { # Co-location
if (p1 < p2) {
cohort1 <- 1 # All the foos
cohort2 <- 0 # No one
dist1 <- abs(x1 - q_citizen(distro, .5))
dist2 <- 0
} else if (p1 > p2) {  
cohort1 <- 0 # No one
cohort2 <- 1 # All the foos
dist1 <- 0
dist2 <- abs(x2 - q_citizen(distro, .5))
} else { # Co-located and equally priced
cohort1 <- .5 # Half the foos
cohort2 <- .5 # The rest of the rabble
dist1 <- abs(x1 - q_citizen(distro, .5))
dist2 <- abs(x2 - q_citizen(distro, .5))
}
}

# Calculate happiness
hap1 <- cohort1 * (10 - 2*(dist1^2) - p1)
hap2 <- cohort2 * (10 - 2*(dist2^2) - p2)
total_hap <- sum(hap1, hap2)

# Calculate profits (acquire currency)
pi1 <- cohort1 * (p1 - .5)
pi2 <- cohort2 * (p2 - .5)

# Fill in data frame
d <- rbind(d, data.frame(p1=p1, p2=p2, type='pi1', pi=pi1, hap=total_hap))
d <- rbind(d, data.frame(p1=p1, p2=p2, type='pi2', pi=pi2, hap=total_hap))
}
}
return(d)
}

We can simulate competition between the two hospitals within the landscape that is visualized in the above heatmaps. The function price_equib below performs this simulation.

price_equib <- function(d) {
	prices <- data.frame(p1 = 2, p2 = 7, pi1 = NA, pi2 = NA, hap1=NA, hap2=NA) # 2 and 7 chosen by random die roll. guaranteed to be random.
	for (i in 1:99) {
	# Find most recent quotes
	p1_price <- prices$p1[nrow(prices)]
	p2_price <- prices$p2[nrow(prices)]
	# Subset available price moves based on quoted prices
	p1_grid <- d[d$type == 'pi1' & d$p2 == p2_price,]
	p2_grid <- d[d$type == 'pi2' & d$p1 == p1_price,]
	#Both now check to see if there is a better price for higher profits, given the current competition
	p1_max_pi <- max(p1_grid$pi) 
	p2_max_pi <- max(p2_grid$pi)
	# They then update their pricing accordingly
	p1_new <- p1_grid$p1[p1_grid$pi == p1_max_pi][1]
	p2_new <- p2_grid$p2[p2_grid$pi == p2_max_pi][1]
	hap1_new <- p1_grid$hap[p1_grid$pi == p1_max_pi][1]
	hap2_new <- p2_grid$hap[p2_grid$pi == p2_max_pi][1]
	# Add prices to dataframe, so many bargains
	prices <- rbind(prices, data.frame(p1=p1_new, p2=p2_new, pi1=p1_max_pi, pi2=p2_max_pi, hap1=hap1_new, hap2=hap2_new))
}
return(prices)
}

Is there a “first-mover” advantage? We can perform more simulations to find out. In this case, we assume that hospital 1 gets to set its price first, with full knowledge of what hospital 2 will do afterwards to optimize its own profits.

first_mover <- function(d) {
	best_price <- data.frame(p1=NULL, pi1=NULL, p2=NULL, pi2=NULL)
	for (p1 in seq(0, 10, price_step)) {
	p2_grid <- d[d$type == 'pi2' & d$p1 == p1,]
	p2_max_pi <- max(p2_grid$pi)
	p2_best <- p2_grid$p2[p2_grid$pi == p2_max_pi][1]
	p1_pi <- d$pi[d$type == 'pi1' & d$p1 == p1 & d$p2 == p2_best]
	best_price <- rbind(best_price, data.frame(p1=p1, pi1=p1_pi, p2=p2_best, pi2=p2_max_pi))
}
return(best_price)
}

We can determine the first-mover advantage for the uniform distribution of customers as well.

p1_first <- first_mover(d)
	ggplot() + geom_path(aes(x=p1_first$p1, y=p1_first$pi1)) +
	ggtitle('First-mover price optimization') +
	xlab(expression(p[1])) +
	ylab(expression(pi[1]))

In this case, hospital 1 will set their price, \(p_{1}\), at 2. In this situation, \(\pi_{1}=\) 0.5625 and \(\pi_{2}=\) 0.7812. This makes intuitive sense because under this distribution of customers and set of locations, neither hospital has an advantage.

Conclusion

In this study, we have successfully modeled several different interactions between competing entities. Obviously their pricing had a large effect on customer loyalty, but their location and the distribution of their customers also played a large role in determining the success of their endeavors. Location can be seen as a proxy for product differentiation too. While the model was seemingly limited to geographic constraints, it is easy to see that some prospective clients my see a luxury brand in the same way that someone could see proximity for substitutable goods. This simulation has shown that if a company can engage certain emotions in a customer, such as desirability, durability, or even design aesthetics, they can overcome some of the rigid limitations of competitive business and increase their company's viability.
